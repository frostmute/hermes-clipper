# Contributing to Hermes Clipper

First off, thanks for taking the time to contribute! Grunt. 🌑⚡

## How Can I Contribute?

### Reporting Bugs
- Use the **Bug Report** template.
- Explain the problem and include steps to reproduce.

### Suggesting Enhancements
- Use the **Feature Request** template.
- Explain why the enhancement would be useful.

### Pull Requests
1. Fork the repo and create your branch from `master`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code follows the existing style.

## Token Sovereignty Guidelines
Any new features MUST adhere to our efficiency mandates:
- Prefer local shell primitives over LLM reasoning where possible.
- Use `VAULT_STRUCTURE.md` for folder lookups.
- Maintain **Caveman Mode** compatibility in all agent prompts.
